# Book-to-Video-Script — 提取内容转视频脚本

## Overview

本技能把**已经提取好的知识内容**（书籍榨取报告、读书笔记、要点清单、原文片段）改写/重组为**视频口播脚本文本**。它不负责读书或提取（那是 book-squeezer 的事），只负责「内容 → 视频脚本」这一段的创作。

核心矛盾：视频要**吸引人**（钩子、节奏、故事），但知识类视频不能**偏离主题或夸大原意**。本技能用一套护栏 + 模板解决这个矛盾：让脚本抓人，但每一句话都能回溯到输入内容。

**两条不可逾越的护栏**（详见 `references/hook-story-bank.md`）：

1. **不偏离主题**：脚本的每一个段落都必须服务于输入内容的「核心论点」（thesis）。禁止为了蹭热点而把内容塞进无关叙事。
2. **不歪曲原意**：钩子可以制造好奇，但不得编造书中没有的数据、结论或作者立场；可添加的故事/类比必须明确标注为「延展/类比」，不与作者原话混淆。

## 输入与前置确认

收到「转视频脚本」请求时，先明确输入与上述参数。若用户未给，按默认值并显式说明假设。

**输入来源（任一）**：
- book-squeezer 生成的 `*榨取报告.md`（最佳，含第 0 层定位、证据强度、案例、残渣）
- 自行粘贴的读书笔记 / 要点清单
- 书籍原文片段或核心章节

**必须确认的参数**（未指定时给默认值）：

| 参数 | 可选值 | 默认 | 说明 |
|------|--------|------|------|
| 视频类型 | 短视频 / 中视频 / 长视频 | 中视频 | 决定结构与时长 |
| 时长 | 秒/分钟 | 短视频 60–90s；中视频 5–8min；长视频 15–25min | 影响信息密度 |
| 平台调性 | 抖音/快手（快节奏）/ B站/YouTube（信息密度）/ 视频号（偏干货+共鸣） | B站/YouTube 风格 | 影响钩子与口播语气 |
| 人称 | 第一人称（我/博主视角）/ 第三人称（解说）/ 对话体 | 第一人称口播 | 影响脚本语气 |
| 受众 | 小白 / 进阶 / 同行 | 进阶泛知识受众 | 影响案例难度 |

若输入是 book-squeezer 报告，直接从「第 0 层：全书内核」提取 thesis，从「知识点深度解析」提取素材，从「案例 / 残渣」提取可讲故事的点——无需重新分析书。

## 工作流程

### Step 1: 提取核心论点（thesis）

从输入中确定 **一句话核心论点**（若输入是报告，即「全书内核」；若是笔记，自己归纳）。这是整支脚本的「不偏题锚点」，后续每个段落都要能回答「这段在支撑 thesis 的哪一点」。

### Step 2: 选定钩子角度（Hook）

从 `references/hook-story-bank.md` 的「钩子库」中选一个**忠于内容**的角度。钩子必须来自内容本身（一个反直觉结论、一个矛盾、一个具体数字、一个场景提问），不得为了吸睛而虚构。

> 例：内容是「长期价值投资先求生存再求回报」→ 钩子可用「为什么最赚钱的投资者，第一件事却是想怎么不亏钱？」（来自书中 Type I 错误框架，真实）。

### Step 3: 设计结构

按视频类型选结构（模板见 `references/script-formats.md`）：

- **短视频（60–90s）**：钩子(5s) → 一个核心观点 + 一个故事/类比(40s) → 反转或金句(15s) → CTA(10s)。只讲透一个点。
- **中视频（5–8min）**：钩子 → 2–4 个知识点（每点 = 观点 + 故事/案例 + 回扣 thesis）→ 复盘/升华 → CTA。三幕式。
- **长视频（15–25min）**：章节式，每章一个子论点，章内复用中视频结构；片头强钩子，片尾总结 + 系列预告。

### Step 4: 编织故事与案例（可控加成）

从输入素材中优先选用书里的真实案例（第 2 层肉质 / 案例）。若需补充故事来「抓人」：

- 只能用**类比 / 延展**（如把抽象概念映射到一个生活场景），并在脚本中用 `[类比]` 标注，不与作者原话混同。
- 补充故事必须**服务 thesis**，禁止离题炫技。
- 不确定是否忠于原意时，宁可不加，或在口播中用「打个比方」明确区分。

钩子与节奏技巧（模式中断、好奇缺口、callback 收尾）见 `references/hook-story-bank.md` 的「吸引力技法」。

### Step 5: 套用脚本格式输出

按 `references/script-formats.md` 的对应模板输出，每段标注：

- `[画面]` / `[镜头]`：给剪辑/拍摄的指示（可简写）
- `[口播]`：念出来的台词
- `[时长]`：该段预估秒数
- `[BGM/音效]`：可选

输出为纯文本 Markdown，文件名 `{书名/主题}-视频脚本-{类型}.md`，保存到当前工作目录。

### Step 6: 护栏自检（必做）

输出前逐项核对（参考 `references/hook-story-bank.md` 的「自检清单」）：

- [ ] 每个段落都能回溯到输入内容的某条信息（不偏题）
- [ ] 钩子未编造书中没有的数据/结论/作者立场
- [ ] 所有「增补故事/类比」已用 `[类比]` 标注，未冒充原书内容
- [ ] 无夸大、无标题党式误导（好奇但不骗）
- [ ] 证据强度得到尊重（🟦 作者观点类内容未包装成铁证）
- [ ] 时长与信息密度匹配所选视频类型
- [ ] CTA 与内容相关，不硬广

### Step 7: 交付与可选导出

默认交付 Markdown 脚本。若用户要 Word/PDF 版，可调用 book-squeezer 的 `export_report.py` 思路，或给出排版建议。本技能不内置导出脚本，保持专注。

## 与 book-squeezer 的边界

- **book-squeezer**：读书 → 结构化知识报告（分析阶段）。
- **book-to-video-script（本技能）**：知识报告 → 视频脚本（表达阶段）。
- 两者**独立**：本技能只读 book-squeezer 的输出，不反向修改它；本技能不重新榨取书，也不做证据分级——它假设输入已是可靠内容，只做「表达层」创作。

## 质量检查清单

- [ ] 已确认视频类型 / 时长 / 平台 / 人称 / 受众（或已说明默认假设）
- [ ] 有清晰的一句 thesis 且全程不偏题
- [ ] 钩子真实、来自内容、不夸大
- [ ] 结构符合所选视频类型模板
- [ ] 故事/类比已标注且服务主题
- [ ] 脚本含 [画面]/[口播]/[时长] 标注，可直接用于拍摄/剪辑
- [ ] 通过 Step 6 护栏自检
- [ ] 文件已保存为 `{主题}-视频脚本-{类型}.md`

## 注意事项

1. **忠诚优先于吸睛**：宁可钩子平一点，也不能编造或歪曲。视频的可持续信任比单条播放量重要。
2. **故事是手段不是目的**：每个故事都要在 30 秒内回扣 thesis，否则删。
3. **尊重证据强度**：输入里的 🟦 作者观点，脚本里不能讲成「研究证明」。
4. **不混入未提取内容**：脚本素材应来自本次输入；要引用输入外的知识，先问用户。
5. **标注增补**：任何输入中没有的举例/类比，必须显式标注，避免以讹传讹。
