# 视频脚本格式模板

本文件提供三类视频脚本的结构骨架与标注规范。所有模板均为 Markdown，便于直接拍摄/剪辑或交给 AI 配音。

## 标注规范

每段用以下标签标注，剪辑/口播一看就懂：

| 标签 | 含义 | 示例 |
|------|------|------|
| `[画面]` | 镜头/画面指示（给拍摄或剪辑） | `[画面] 黑屏，一行白字浮现` |
| `[口播]` | 念出来的台词（纯文案，不含标注） | `[口播] 你有没有想过，最会赚钱的人……` |
| `[时长]` | 该段预估秒数 | `[时长] 5s` |
| `[BGM/音效]` | 背景音乐或音效提示（可选） | `[BGM] 悬疑低音渐入` |
| `[类比]` | 标注这是脚本增补的类比/故事，非原书内容 | `[类比] 就像给手机充电，低于 20% 就焦虑` |

> 规则：所有输入内容里**没有**的举例、类比、生活场景，一律加 `[类比]` 前缀，绝不冒充原书结论。

---

## 一、短视频脚本（60–90s，讲透 1 个点）

结构：**强钩子 → 1 个核心观点 + 1 个故事/类比 → 金句反转 → CTA**

```markdown
# 《[主题]》短视频脚本（[平台]风 / 约 75s）

[画面] 黑屏，大字标题弹出：[一句钩子式提问]
[口播] [钩子：5s，制造好奇，必须来自内容]
[时长] 5s
[BGM] 前 3 秒静音，第 4 秒节奏点切入

[画面] 博主出镜 / 关键概念字卡
[口播] [核心观点：用大白话讲清这一个点，20s]
[时长] 20s

[画面] 切换生活化画面 / 动画示意
[口播] [故事或类比：把观点落到具体场景，40s]
[时长] 40s
[类比] （若是增补故事，在此标注）

[画面] 回到博主 / 金句定格
[口播] [反转或金句：把观点收成一个让人记住的句子，10s]
[时长] 10s

[画面] 结尾卡片：关注 + 下期预告
[口播] [CTA：关注 / 评论区聊聊 / 下期讲 X，10s]
[时长] 10s
```

**短视频铁律**：只讲 1 个点，钩子=观点预告，故事必须 30 秒内回扣观点。

---

## 二、中视频脚本（5–8min，2–4 个知识点）

结构：**三幕式**：钩子 → 主体（每点 观点+故事+回扣）→ 升华 → CTA

```markdown
# 《[主题]》中视频脚本（[平台]风 / 约 6min）

## 片头 Hook（0:00–0:30）
[画面] [强视觉开场]
[口播] [钩子：用内容里最反直觉的一点开场，不剧透结论]
[时长] 30s
[BGM] 主旋律进入

## 第一幕：立论（0:30–1:30）
[口播] [抛出 thesis：这支视频想说清楚的一件事]
[画面] 论点字卡
[时长] 60s

## 第二幕：主体（1:30–4:30）
### 知识点 1：[概念名]
[画面] [对应画面]
[口播] [观点：大白话定义 + 为什么重要，45s]
[口播] [故事/案例：书里真实案例 或 [类比] 增补类比，60s]
[口播] [回扣 thesis：这点如何支撑总论点，15s]
[时长] 120s

### 知识点 2：[概念名]
[同上结构，每个 90–150s]

### 知识点 3–4（按需）

## 第三幕：升华与收尾（4:30–6:00）
[口播] [把几个点串成一句话行动建议 / 认知升级]
[画面] 总结字卡
[时长] 60s
[口播] [CTA：关注 + 评论区提问 + 下期预告]
[时长] 30s
```

**中视频铁律**：每点严格「观点→故事→回扣 thesis」，避免点与点之间失去主线。

---

## 三、长视频脚本（15–25min，章节式）

结构：**强片头钩子 → N 个章节（每章复用中视频三幕）→ 全局总结 + 系列钩子**

```markdown
# 《[主题]》长视频脚本（深度解读 / 约 20min）

## 片头（0:00–1:00）
[口播] [全书级钩子：这本书/这个主题最反直觉的真相]
[画面] 书封 / 关键数据
[时长] 60s

## 第一章：[子论点 A]（1:00–6:00）
### 1.1 背景
### 1.2 核心机制（观点+故事+回扣）
### 1.3 一个反例或边界（来自残渣层，增加可信度）
[章节时长] ~5min

## 第二章：[子论点 B]（6:00–12:00）
[同上]

## 第三章：[子论点 C]（12:00–18:00）
[同上]

## 结尾（18:00–20:00）
[口播] [全局复盘：三章如何构成完整认知]
[口播] [CTA + 系列下集预告（若做成合集）]
[时长] 120s
```

**长视频铁律**：章与章之间用「上一章留下的疑问」做 transition（callback），让长内容不散。

---

## 时长分配速算

| 类型 | 总时长 | 钩子 | 主体占比 | CTA |
|------|--------|------|----------|-----|
| 短视频 | 60–90s | 5–10s | ~75% | 10s |
| 中视频 | 5–8min | 30s | ~80% | 60–90s |
| 长视频 | 15–25min | 60s | ~85% | 120s |

> 节奏提示：每 20–40s 必须有一次「信息转换」（换画面 / 抛新问 / 给例子），否则观众流失。
